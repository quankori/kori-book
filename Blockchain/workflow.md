# Blockchain Workflow

- Node is defined as a connection point.

- Node will create and hash transaction when user need interact data on blockchain (example: smart contract).

- User sign and node will send transaction to network P2P

- Now transaction is unconfirmed transaction and save on memory node because it's not confirmed by other node to enter blockchain.

- Process use consensus algorithm choose the winner node, the winner node will collect unconfirmed transaction to one block and add to end of blockchain on memory node.

- After winner node added to end of blockchain, it will deploy that blockchain to network P2P. Other node in network will be verify new block and clone blockchain to update memory.
